# schemas

JSON Schemas for pipeline artifacts.

## Files

### input_spec.schema.json
Purpose: validate the raw YAML/JSON specification.
Inputs: used by ingest/validate_and_normalize.py.
Outputs: list of validation errors when spec is invalid.

### normalized_spec.schema.json
Purpose: target shape of normalized_spec.json.
Inputs: used by validators and tests.
Outputs: ensures normalized spec is consistent.

### io_map.schema.json
Purpose: schema for io_map.json generated by analyze/extract_io.py.
Inputs: used by validators and downstream tools.
Outputs: ensures entity IO structure is stable.

### program_plan.schema.json
Purpose: schema for program_plan.json generated by analyze/plan_programs.py.
Inputs: used by validators and generators.
Outputs: ensures program list and procedures are consistent.

### compile_report.schema.json
Purpose: schema for compile reports produced by compile/run_compile_suite.py.
Inputs: used by validation/reporting.
Outputs: enforces compile report shape.

### diagnosis.schema.json
Purpose: schema for diagnostic artifacts (autofix).
Inputs: used by validate/autofix modules.
Outputs: enforces diagnosis structure.

### patch_plan.schema.json
Purpose: schema for patch plans used by autofix.
Inputs: used by validate/autofix modules.
Outputs: enforces patch plan structure.

### __init__.py
Purpose: package marker (no runtime logic).
